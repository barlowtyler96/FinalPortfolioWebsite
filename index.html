MY WORK
CulinaryShares Blazor Server
Technologies used:
Bootstrap 5, Azure AD B2C, OAuth 2.0, OpenID Connect, Razor Components
This is a Blazor Server application that I made to consume the Recipes Web API. The app calls the Recipes Web API after obtaining an access token on behalf of a user. The app uses Azure AD B2C to allow users to create an account and sign in. Once a user is authenticated, they are authorized to view a page where they can submit recipes for review. I made a short video showing off the project.

CulinaryShares Web API
Technologies used:
Azure AD, OAuth 2.0, SQL Server, Dapper, Postman
This solution consists of two projects, a class library for data access and a web API. My data access library utilizes dapper with stored procedures. The library is referenced in my API. The API is then consumed by my blazor server app. My favorite feature is the server-side paging. When a user searches for a recipe on the blazor app, only the results for the current page are returned and stored in memory.
